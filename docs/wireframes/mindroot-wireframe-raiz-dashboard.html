Raiz Dashboard — Decisões de UX

D67: Health score = ring com gradiente do companion. Mostra "X/9 domínios ativos". Não é percentagem — é contagem de domínios que passaram do estágio 1. Sub-info mostra highlights (melhor domínio, regressão).
D68: Domain rows = lista vertical (não grid 3×3). Mais legível em mobile, permite status descritivo. Cada row tem: icon, nome, status text, barra de 7 segmentos (estágios), e geometria atual. Module como borda esquerda.
D69: Stage bar = 7 segmentos horizontais. Preenchidos com cor do estágio correspondente. Regressão mostra segmentos em vermelho desvanecendo. Dá leitura instantânea de "onde está" e "por onde passou".
D70: Regressão = alerta visual forte. Row com fundo rosa, borda vermelha, badge "↓ regrediu". Alert card acima da lista com contexto ("fotos sem backup há 3 semanas") e CTA "abrir sessão →". O sistema não esconde a regressão — mostra e convida gentilmente a resolver.
D71: Domain detail = tela separada (push navigation, não modal). Jornada visual com 7 nós circulares conectados por linhas. Nó atual tem anel de foco. Items listados com geometria individual. Gate pro próximo estágio explicado em texto.
D72: "Próximo passo" = card companion com avatar ○, texto do próximo passo concreto, e seta →. Tap abre companion pra sessão focada naquele domínio. Sempre concreto, nunca genérico.
D73: Rituals de manutenção = seção permanente no dashboard. Semanal (inbox zero), mensal (review), trimestral (audit completo). Checkmarks clicáveis. Frequência como chip. Items do Raiz type=habit com extension Recurrence.
D74: Raiz Dashboard NÃO é tab no BottomNav. Acessível via: (a) seção "raiz" na Home, abaixo dos items, ou (b) Settings → "minha raiz", ou (c) após onboarding, primeiro lugar onde o app abre. Decisão final: seção scrollável na Home, com card "ver todos os domínios →" que abre a tela full.

Localização na navegação

  • Home: mini-card "raiz" com health ring + pior domínio + "ver tudo →"
  • Full dashboard: tela pushada da Home (não tab)
  • Domain detail: tela pushada do dashboard
  • Session: companion full-height (mesmo do onboarding)

Dados de cada domínio

  • Estágio atual = média ponderada dos genesis_stage dos items com tag #domain:*
  • Items = query: items WHERE tags CONTAINS '#domain:identity' ORDER BY genesis_stage DESC
  • Regressão = detectada quando estágio médio cai entre audits (query automática)
  • Gate = configurado por domínio (pode ser tabela ou JSON config)
  • Próximo passo = gerado pelo companion com contexto do domínio
