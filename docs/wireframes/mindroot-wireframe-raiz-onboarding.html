Raiz Onboarding — Decisões de UX

D59: Welcome screen = sem formulário, sem login, sem setup. Começa com filosofia: "uma gaveta por vez". Símbolo = anéis concêntricos com geometria do ponto (·) no centro → de dentro pra fora. "Depois, talvez" como escape respeitoso.
D60: 3 portas de entrada como cards verticais. Cada uma com cor, emoji, e tag descritivo ("pra quem precisa de momentum"). "Me guia" é a porta padrão (highlighted, badge "recomendado"). Nenhuma porta é errada.
D61: Sessão de inventário = companion full-height (não bottom sheet). É o modo principal. Header mostra domínio + estágio. Progress dots mostram 9 domínios (1 ativo, rest pending). O companion guia com perguntas abertas, captura items em tempo real como cards inline.
D62: Captured items card = lista inline no chat. Mostra items capturados com geometria (·), título, e type chip (resource, doc, etc). Atualiza conforme o usuário despeja. O companion classifica silenciosamente — o usuário vê o resultado, não o processo.
D63: Soul checkpoints = cards com borda rosa, texto em itálico, label "soul checkpoint · opcional". Aparecem em 5 momentos: início, achado emocional, decisão difícil, vitória, fadiga. Quick replies incluem "pesado... vamos pausar" que é respeitado imediatamente.
D64: Panorama = grid 3×3 dos 9 domínios com geometria de progresso. Domínios não iniciados ficam dimmed (opacity 0.4). Domínios com items mostram contagem. O ativo (último inventariado) tem borda highlight. Summary card mostra totais + próximo sugerido.
D65: Pausar é sempre válido. Botão "pausar" é igual em peso visual ao "continuar". Não existe pressão. "A geometria espera." Progresso fica salvo. O panorama reaparece quando o usuário volta.
D66: Formatos de sessão controlados pela duração natural. O companion observa: 10 min sem interação → "quer pausar aqui? tá salvo." Não existe timer visível. O tempo aparece só no summary (panorama).

Flow completo

  • 1. Welcome → "começar"
  • 2. 3 portas → escolhe uma
  • 3. Companion abre no domínio selecionado
  • 4. Despejo livre → companion captura + classifica
  • 5. Soul checkpoints em momentos emocionais
  • 6. Panorama → espelha o que foi capturado
  • 7. "Continuar" → próximo domínio / "Pausar" → salva e sai
  • 8. Após todos os domínios: Raiz Dashboard como feature permanente

Onde vive na navegação

  • Primeiro acesso: onboarding intercepts antes da Home
  • Depois: Raiz Dashboard acessível via Home (seção "raiz" abaixo dos items) ou via Settings
  • Sessões de domínio: tap no domínio no dashboard → abre companion no domínio
  • NÃO é uma tab no BottomNav — é feature, não seção
